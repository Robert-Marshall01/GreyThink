# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in GreyThink, please follow these steps:

1. **Do Not Create a Public Issue** for security risks.
2. **Contact the Repository Owner Directly:**  
   Email: (use the email in the GitHub profile if available) or open a private discussion if possible.
3. **Include Details:**  
   Provide as much information as possible to help reproduce and understand the vulnerability.

## Supported Versions

This is an individual project with diverse programs. Security updates may be provided on a best-effort basis.

## Responsible Disclosure

Please allow the repository owner reasonable time to address reported vulnerabilities before disclosing them publicly.

Thank you for helping keep GreyThink safe!
